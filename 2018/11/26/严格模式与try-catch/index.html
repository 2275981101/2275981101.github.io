严格模式与try-catch

Posted on 2018-11-26 | Edited on 2018-12-18

ES5.0严格模式

• 浏览器 基于的es3.0的 + es5.0的新增方法 使用的
• es3.0 和 es5.0产生冲突的部分
• es5.0严格模式: 那么es3.0和es5.0产生冲突的部分就是用es5.0，否则会用es3.0

es5.0严格模式的启动: 必须写在页面逻辑的最顶端 或 方法里面最顶端(局部)

"use strict";

局部使用
function test(){
    "use strict";
    console.log(arguments.callee);
}
test();

严格模式下: • with, arguments.callee, caller(不能使用)
• 变量赋值必须声明,局部this必须赋值
• 拒绝重复属性和参数
• es3.0 都不能使用eval();

如:
function test(name, name){
    "use strict";
    console.log(name);
}
test(1,2);

var obj = {
    name : '123',
    name : '234'
}

Error.name的六种值对应的信息

1.EvalError: eval()的使用与定义不一致
2.RangeError: 数值越界
3.ReferenceError: 非法或不能识别的引用数值
4.SyntaxError: 发生语法解析错误
5.TypeError: 操作数类型错误
6.URIError: URI处理函数使用不当
